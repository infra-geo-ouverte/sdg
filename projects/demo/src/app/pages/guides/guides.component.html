Cette documentation est dédié aux développeurs

Cette bibliothèque est basé sur @angular/material. Ce choix a été fait notamment pour la grande qualité d'accessibilité offerte aux utilisateurs, ainsi que pour la cohérence et la robustesse de ses composants d'interface.

Les composants sont développés pour prendre en compte l'accessibilité et pour la majorité compatible avec le "server side rendering"

Tokens CSS

Les composants sont basés sur les variable CSS défini ci-dessous. Les variables permettent d'améliorer la maintenabilité en stylant différent aspect couleurs, la typographie, l'espacement et la disposition de manière globale. Ces tokens permettent d'assurer la cohérence du design et une personnalisation facile. La manière la plus simple de voir l'ensemble des variables CSS est d'ajouter le style et d'inspecter le style de votre application car ces variables sont générés dynamiquement lors de l'improtation.

  • Couleurs :

    --sdg-color-NOM_COULEUR-VARIANTE

    NOM_COULEUR : blue|grey|pink|red|green|yellow|violet|white|black *white et black ne continennent pas de variente
    VARIANTE :pale|light|normal|medium|dark

    Consulter les couleurs de SDG

  • Palettes : --sdg-palette-NOM_COULEUR-GRADIENT

    Les gradients sont cotés de 100 à 900

  • Typographie :,
    • --sdg-font-size-*
    • --sdg-line-height-*
    • --sdg-title-margin-NIVEAU_TITRE-(mt|mb) mt: margin-top et mb: margin-bottom

      NIVEAU_TITRE : h1|h2|h3|h4|h5|h6

  • Élévation : --sdg-elevation-*
  • Disposition :
    • --sdg-spacer-*
    • --sdg-max-content-width

Vous pouvez référencer ces tokens dans vos propres styles pour une intégration complète au thème.

Style de mise en page

En dehors des normes SDG, un sous-ensemble de classes pour la mise en page provenant de Bootstrap est disponible. Importer le style dans le fichier style.scss

Règles de filtrage (dans l'objectif d'optimisation de la taille applicative) :
  • .container
  • .col
  • .row
  • .d-flex
  • .flex

Règles de filtrage : seules les règles CSS dont les sélecteurs contiennent un de ces motifs sont incluses dans les styles distribués.

Concept et configuration de l'application

Pour configurer votre application avec Angular, vous pouvez utiliser les outils et interfaces fournis par @igo2/sdg.

Routing

Nous réutilisons le Routing d'Angular mais avec quelques distinctions qui bénéficieront à quelques composants. Nous proposons une interface personnalisée SdgRoutes pour définir les routes de l'application. Cette interface est totalement optionnelle et étend la configuration standard d'Angular Router et permet d'ajoute des propriétés utiles pour certains composants (ex. : fil d'Ariane).

Par exemple, la propriété isHome permet d'indiquer la page d'accueil, et hidden permet de masquer une route dans la navigation principale ou le fil d'Ariane.

Internationalisation (i18n)

Basé sur la librairie ngx-translate, plusieurs fonctionnalités sont offertes pour faciliter la gestion de la traduction et des libellés des composants. Ce module est optionnel, il est possible d'utiliser la stratégie d'internationalisation qui convient le mieux à vos besoins

provideTranslation

Le provider provideTranslation permet de configurer l'internationalisation de l'application. Il offre plusieurs fonctionnalités :

  • withRouterTitleResolver : synchronise le titre de la page avec la route active.
  • withLanguageFromUrl : détecte la langue à partir de l'URL.
  • withWaitOnI18nReady : attend que les traductions soient chargées avant d'afficher l'application.
