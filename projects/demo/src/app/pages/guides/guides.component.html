Variables CSS

Les composants sont basés sur diverses variables CSS définies ci-dessous. Ces variables permettent d'améliorer la cohérence des styles et de personnaliser facilement les couleurs, la typographie, l'espacement, etc.

La manière la plus simple de voir l'ensemble des variables CSS est d'ajouter le style SDG puis d'inspecter les styles de votre application, car ces variables sont générées dynamiquement lors de l'importation.

  • Couleurs :

    --sdg-color-NOM_COULEUR-VARIANTE

    NOM_COULEUR : blue|grey|pink|red|green|yellow|violet|white|black
    * white et black ne possèdent pas de variantes.
    VARIANTE : pale|light|normal|medium|dark

  • Palettes :

    --sdg-palette-NOM_COULEUR-GRADIENT

    Les gradients sont cotés de 100 à 900.

  • Typographie :
    • --sdg-font-size-*
    • --sdg-line-height-*
    • --sdg-title-margin-NIVEAU_TITRE-(mt|mb) mt: margin-top mb: margin-bottom

      NIVEAU_TITRE : h1|h2|h3|h4|h5|h6

  • Élévation : --sdg-elevation-*
  • Disposition :
    • --sdg-spacer-*
    • --sdg-max-content-width

Vous pouvez référencer ces variables dans vos propres styles pour une intégration complète au thème.

Concept et configuration de l'application

Pour configurer votre application avec Angular, vous pouvez utiliser les outils et interfaces fournis par @igo2/sdg.

Routing

Le routing d'Angular est réutilisé mais avec quelques distinctions et particularités qui viennent bonifier quelques composants. Une interface personnalisée SdgRoutes permet de définir les routes de l'application. Cette interface optionnelle étend la configuration standard d'Angular Router et permet d'ajouter des propriétés utiles pour certains composants :

Par exemple, la propriété isHome permet d'indiquer la page d'accueil, et hidden permet de masquer une route dans le menu de navigation principale ou dans le fil d'Ariane.

Internationalisation (i18n)

Basée sur la librairie ngx-translate, plusieurs fonctionnalités sont offertes pour faciliter la gestion de la traduction et des libellés des composants. Ce module est optionnel et il est possible d'utiliser la stratégie d'internationalisation qui convient le mieux à vos besoins.

provideTranslation

Le provider provideTranslation permet de configurer l'internationalisation de l'application. Il offre plusieurs fonctionnalités :

  • withRouterTitleResolver : synchronise le titre de la page avec la route active.
  • withLanguageFromUrl : détecte la langue à partir de l'URL de la page courante.
  • withWaitOnI18nReady : attend que les traductions soient chargées avant d'afficher l'application.
